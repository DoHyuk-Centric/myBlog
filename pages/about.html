Introduction

Front-End Developer

개념보다 동작을 기준으로 이해합니다.

문법 설명보다 브라우저에서 실제로 어떻게 렌더링·동작하는지를 먼저 확인합니다.

프로젝트 과정에서 발생하는 문제를 결과 → 원인 → 구조 순서로 분석합니다.

“왜 이렇게 동작하는가”를 기준으로 코드를 수정합니다.

UI/UX 관점의 프론트엔드 사고를 지향합니다.

다크모드, 모달, 드롭다운, 햄버거 메뉴 등 사용자 인터랙션 요소를 직접 구현하며 구조를 이해합니다.

단순한 레이아웃 구현보다 사용 흐름과 시각적 피드백을 중요하게 생각합니다.

실제 서비스(UI)를 분석하고, 이를 HTML/CSS/JS로 재구현하려는 방식을 사용합니다.

협업을 매우 중요하게 생각하며, 다른 영역의 방식도 이해하려고 노력하고 공부합니다.

프론트엔드 관점에 국한되지 않고, 백엔드 API 흐름과 역할을 함께 고려합니다.

React, 순수 HTML/JS, 서버 통신 방식의 차이를 비교하며 기술 선택의 이유를 이해하려 합니다.

다른 영역의 구현 방식을 파악해 의사소통 비용을 줄이는 개발자를 지향합니다.

문제해결 능력을 키우기 위해 노력하고 있습니다.

오류가 발생하면 즉시 해결책을 찾기보다 재현 → 원인 파악 → 해결 과정을 거칩니다.

폰트, SVG, Tailwind utility, CSS 효과 등 사소해 보이는 문제도 원인을 명확히 하려는 습관이 있습니다.

“일단 되게 만드는 것”보다 왜 되는지 설명할 수 있는 상태를 목표로 합니다.
